# Desktop Responsive Design

**Дата:** 2026-04-03
**Статус:** утверждён

## Задача

Добавить адаптивность для десктопных экранов (1024px+) к существующему мобильному свадебному лендингу.

## Подход

Один `@media (min-width: 1024px) { ... }` блок в конце тега `<style>` в `index.html`. Мобильные стили остаются нетронутыми.

## Дизайн по секциям

### Глобальные контейнеры
- Все секции получают внутренний `max-width: 1200px; margin: 0 auto` на контентных блоках
- Предотвращает растягивание текста на широких мониторах

### Hero
- Фото остаётся fullwidth
- `.hero-content` ограничивается `max-width: 900px; margin: 0 auto`
- `border-radius` нижних углов уменьшается с `33%` до `10%` (на мобиле 33% создаёт слишком глубокую кривую)
- Шрифты увеличиваются через `clamp()` максимумы

### Countdown
- `aspect-ratio: 1/1` сохраняется
- `max-width: 650px; margin: 0 auto` — центрированный квадратный виджет
- `.countdown-number` вырастает до `4rem`
- `padding-top` пересчитывается под фиксированную ширину

### Timeline (расписание)
- Горизонтальный лейаут: `flex-direction: row; justify-content: center`
- Иконка сверху → горизонтальная линия → точка → время → название
- `max-width: 900px` на `.schedule-list`
- `.timeline-spine::before` перестраивается в горизонтальную линию

### Venue
- Остаётся центрированной колонкой
- Фотографии: `width: 35%` (вместо 50%) для лучших пропорций
- `max-width: 800px` на контенте

### Details и прочие секции
- `max-width: 800px; margin: 0 auto` на внутреннем контенте

## Что НЕ меняется
- Цветовая схема
- Типографика (семейства шрифтов)
- Мобильные стили (всё ниже 1024px)
- Анимации при скролле
- TypeScript-логика
